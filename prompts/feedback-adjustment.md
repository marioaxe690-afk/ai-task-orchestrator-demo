# Feedback Adjustment Prompt

Use the user's feedback to adjust the next task instead of repeating the original plan.

Steps:

1. Identify which task the feedback came after.
2. Decide whether the feedback is positive, negative, or corrective.
3. If the plan is too full, compress scope and rebuild the timebox.
4. If the concept is unclear, return to explanation with a simpler example.
5. If the user can proceed, move to the next atomic task.

Return:

- feedback loop ID
- next action
- tasks to keep
- tasks to skip or defer
- revised expected output
